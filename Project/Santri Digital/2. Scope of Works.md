# ðŸ“Œ 1. Project Overview

Proyek ini bertujuan untuk membangun aplikasi web sederhana yang digunakan untuk mengelola pembayaran SPP pondok pesantren. Sistem akan memungkinkan:

1. Admin membuat/mengatur akun santri
2. Admin membuat tagihan bulanan untuk seluruh santri
3. Admin memverifikasi pembayaran santri
4. Santri menggunakan mobile app untuk upload bukti
5. Santri melihat status pembayaran dari aplikasi mobile

sistem dirancang dengan arsitektur **web-admin + mobile-app**:

- **Santri sepenuhnya menggunakan aplikasi mobile (Android)** untuk melihat tagihan dan mengunggah bukti pembayaran.
- **Admin menggunakan dashboard web** untuk mengelola santri, membuat tagihan, dan memverifikasi pembayaran.

# ðŸ“Œ 2. Project Scope

Ruang lingkup dibagi menjadi dua platform utama: **Admin Web** dan **Mobile App Santri**, serta modul sistem pendukung.

---

## 2.1. Admin Scope

**A. Authentication**
- Admin login menggunakan username & password
- Session/token-based authentication

**B. Manajemen Santri**
- Lihat daftar semua santri
- Tambah santri manual
- Edit data santri
- Reset password santri
- Nonaktifkan santri (opsional)

**C. Import Data Santri**
- Upload CSV atau Excel untuk menambah santri secara massal
- Validasi kolom, format, duplikasi username
- Preview data sebelum import
- Summary sukses/gagal

**D. Manajemen Tagihan SPP**
- Set nominal tagihan SPP untuk seluruh santri sekaligus (mass billing)
- Mencegah duplikasi tagihan bulanan
- Override nominal untuk santri tertentu (opsional)
- Lihat daftar tagihan + status pembayaran

**E. Verifikasi Pembayaran**
- Melihat bukti pembayaran santri
- Menyetujui pembayaran
- Menolak pembayaran (dengan alasan opsional)
- Status otomatis diperbarui

**F. Dashboard Admin**
- Ringkasan jumlah santri
- Ringkasan status pembayaran
- Tagihan bulan berjalan

---

## 2.2. Santri Scope

**A. Authentication**
- Login santri menggunakan akun yang dibuat admin

**B. Pembayaran / Tagihan**
- Lihat tagihan bulan berjalan
- Lihat nominal, batas pembayaran, dan status
- Lihat riwayat pembayaran sebelumnya

**C. Upload Bukti Pembayaran**
- Upload foto dari kamera
- Upload dari galeri
- Re-upload jika pembayaran ditolak
- Preview gambar sebelum upload

**E. UI/UX Mobile**
- Desain sederhana dan mudah digunakan

---

## 2.3. System Scope

**A. Backend API (Golang)**
- Autentikasi admin & santri
- Endpoint untuk data santri, tagihan, pembayaran
- Upload bukti pembayaran
- Dashboard admin

**B. Database**
- Tabel Users
- Tabel Students
- Tabel Bills
- Tabel Payments
- Tabel PaymentProofs

**C. File Management**
- Penyimpanan bukti pembayaran di server VPS
- Struktur folder terorganisir per bulan

**D. Deployment**
- VPS + Docker
- Setup domain & SSL
- CI/CD sederhana (opsional)

**E. Mobile App Build**
- Build APK
- Optional: Publish ke Google Play Store (biaya akun ditanggung client)
