## 2.1. Performance Requirements

NFR-01

Sistem harus dapat menampung jumlah santri 100â€“500 tanpa penurunan performa signifikan.

NFR-02

---

## 2.2. Usability Requirements

NFR-10

Sistem harus responsif dan bisa digunakan di HP santri (Android/iOS).

NFR-11

UI sederhana, mudah dipahami, tanpa navigasi kompleks.

NFR-12

Error messages harus jelas, termasuk:

Validasi form

Upload file gagal

Password salah

Import gagal



---

## 2.3. Security Requirements

NFR-20

Password harus disimpan menggunakan hashing (BCrypt/Argon).

NFR-21

Upload file harus divalidasi:

Hanya JPG/PNG

Maksimum ukuran tertentu

Scan jenis file (MIME detection)


NFR-22

Santri hanya dapat mengakses data miliknya sendiri (authorization).

NFR-23

Admin hanya dapat diakses role admin.

NFR-24

Token/session harus aman dan tidak mudah dipalsukan.


---

## 2.4. Reliability & Availability

NFR-30

Sistem harus memiliki uptime minimal 95% (standar aplikasi kecil di VPS).

NFR-31

Sistem harus mampu melakukan recovery dari kerusakan file upload (misal file corrupt).

NFR-32

Backup database minimal 1 kali per minggu (manual/otomatis tergantung deployment).


---

## 2.5. Maintainability & Scalability

NFR-40

Codebase harus modular dan mudah dikembangkan (backend dan frontend terpisah).

NFR-41

Sistem harus dapat ditingkatkan ke:

Fitur laporan bulanan

Fitur penggajian

Dashboard grafik Tanpa mengubah struktur inti database secara ekstrem.


NFR-42

Dokumentasi dasar harus disediakan:

Cara menambah santri

Cara membuat tagihan bulanan

Cara verifikasi pembayaran



---

## 2.6. System Constraints

NFR-50

Sistem hanya berbasis web, tanpa aplikasi mobile native.

NFR-51

Desain UI sederhana (no complex animations, no heavy libraries).

NFR-52

Server target: VPS kecil (1 vCPU, 1GB RAM).
